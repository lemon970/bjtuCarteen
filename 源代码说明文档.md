# 源代码说明文档

本文档用于说明当前项目目录下各个子目录、源代码文件和其他配套文件的用途，便于提交、答辩和后续维护。

## 1. 项目整体说明

本项目是一个基于 Spring Boot 的食堂仿真后端系统，核心功能包括仿真参数接收、离散事件仿真执行、报告生成、报告分页读取、异步任务提交、SSE 状态推送、批处理对比接口以及基础前端演示页面。

当前主要技术栈如下：

| 类型 | 内容 |
| --- | --- |
| 开发语言 | Java 17 |
| 后端框架 | Spring Boot 3.3.4 |
| Web 接口 | Spring MVC |
| 参数校验 | Spring Validation |
| JSON 序列化 | Jackson |
| 构建工具 | Maven |
| 测试框架 | JUnit 5、Spring Boot Test、MockMvc |

## 2. 子目录说明

## 4. 控制层文件说明

| 文件 | 说明 |
| --- | --- |
| `controller/SimulationController.java` | 核心 API 控制器。提供同步仿真 `/api/simulation/run`、异步仿真 `/api/simulation/run/async`、任务状态查询、SSE 状态流、批处理对比 `/api/simulation/optimize`、报告读取、报告列表、timeline/history 分页读取等接口。 |
| `controller/GlobalExceptionHandler.java` | 全局异常处理器。统一处理参数校验失败、请求体格式错误、非法参数、非法状态和未知异常，并包装为统一的 `ApiResponse` 返回结构。 |

## 5. DTO 文件说明

| 文件 | 说明 |
| --- | --- |
| `dto/ApiResponse.java` | 统一接口响应包装类，返回结构为 `code`、`message`、`data`，用于保持前后端接口格式一致。 |
| `dto/SimConfig.java` | 仿真配置对象，定义窗口数量、座位数量、学生数量、到达率、服务时间、随机种子、打包概率、下课波峰、天气影响、分布配置、桌位配置等输入参数。 |
| `dto/OptimizationRequest.java` | 批处理对比接口请求对象，支持传入基准配置、多个待评估配置、目标指标、约束、最大候选数和返回数量等字段。当前接口定位为显式多配置批处理对比，不做复杂自动搜索。 |
| `dto/SimulationReport.java` | 完整仿真报告对象，包含报告版本、报告 ID、有效随机种子、归一化后的配置、统计 summary、生成时间等信息。 |
| `dto/SimulationSummary.java` | 仿真统计汇总对象，包含历史事件、分钟级 timeline、到达/放弃/服务/堂食/打包/离开人数、等待时间、移动时间、队列峰值、座位利用率、窗口吞吐量、排队论参考指标等统计字段。 |
| `dto/SimulationResult.java` | 仿真历史事件快照对象，记录某一事件发生后的时间、窗口队列、座位占用、累计计数、事件消息、窗口服务数、桌位快照等细粒度回放数据。 |
| `dto/SimulationTimePoint.java` | 分钟级时间线快照对象，用于前端图表展示。相比 `SimulationResult` 更适合按分钟渲染趋势图和大盘。 |
| `dto/QueueTheoryMetrics.java` | 排队论参考指标对象，记录模型类型、服务台数量、到达率、服务率、系统利用率、期望排队长度、期望等待时间等理论参考值。 |

## 6. 仿真引擎文件说明

| 文件 | 说明 |
| --- | --- |
| `engine/BaseEvent.java` | 所有仿真事件的抽象基类，定义事件时间、优先级、事件排序规则和 `process` 执行入口。 |
| `engine/SimulationEngine.java` | 仿真核心引擎。维护事件优先队列、随机数生成器、系统状态、学生对象、统计指标和仿真历史，负责调度并执行全部事件。 |
| `engine/CanteenState.java` | 食堂运行状态对象，维护窗口队列、座位占用、桌位区域，并提供排队、离队、占座、释放座位等状态操作。 |
| `engine/CanteenWindow.java` | 食堂窗口模型，记录窗口 ID、窗口名称和当前队列长度。 |
| `engine/StudentArriveEvent.java` | 学生到达事件，负责注册学生、记录到达人数，并根据移动时间和排队条件进入后续事件。 |
| `engine/MovementEvent.java` | 学生移动事件，显式模拟学生从入口到窗口、从窗口到座位的移动时间。 |
| `engine/ServiceFinishEvent.java` | 服务完成事件，负责处理窗口出队、服务人数统计、堂食/打包决策以及后续找座或离开流程。 |
| `engine/StudentLeaveEvent.java` | 学生离开事件，负责堂食完成后的座位释放和离开人数统计。 |

## 7. 领域模型文件说明

| 文件 | 说明 |
| --- | --- |
| `model/Student.java` | 学生实体，记录学生 ID、打包偏好、排队耐心、窗口偏好、找座耐心、到达类型、群组人数、当前状态、到达/开始移动/完成服务/离开时间和座位分配等信息。 |
| `model/StudentState.java` | 学生状态枚举，用于表示学生在仿真过程中的生命周期状态，例如排队、服务、找座、就餐、离开等。 |
| `model/ArrivalGroup.java` | 到达类型枚举，用于区分普通到达、下课波峰到达、天气影响到达等学生来源。 |
| `model/DiningArea.java` | 就餐区模型，负责桌位构建、单人/多人占座、座位释放、桌位利用时长统计和桌位快照生成。 |
| `model/TableSnapshot.java` | 桌位快照对象，记录某张桌子的容量、占用座位数、空座数、占用时长、座位占用时长和利用率。 |

## 8. 服务层文件说明

| 文件 | 说明 |
| --- | --- |
| `service/SimulationRunService.java` | 同步仿真运行服务。负责配置归一化、创建引擎、调度到达事件、执行仿真、构建 summary 和生成完整报告。 |
| `service/SimulationConfigNormalizer.java` | 配置归一化和校验服务。负责补齐默认值、处理分布配置、校验窗口数量、打包窗口数量、服务时间倍率、概率范围、分页边界等约束。 |
| `service/SimulationArrivalScheduler.java` | 到达事件调度服务。根据总时长、到达率、下课波峰、天气影响、到达分布等配置向引擎中写入学生到达事件。 |
| `service/SimulationTimelineBuilder.java` | 时间线构建服务。将事件级 `history` 压缩/转换为分钟级 `timeline`，供前端图表和分页读取使用。 |
| `service/SimulationReportRepository.java` | 报告文件仓库。负责将仿真报告写入本地 JSON 文件，读取 latest 报告、按 ID 查询报告、生成报告列表，并进行 report_id 安全校验。 |
| `service/SimulationTaskService.java` | 异步任务服务。负责提交后台仿真任务、维护任务状态、执行任务、返回任务快照，并通过 SSE 推送任务状态。 |
| `service/SimulationTaskRecord.java` | 异步任务记录对象，保存任务 ID、报告 ID、提交时间、运行状态、完成时间、错误信息和最终报告。 |
| `service/OptimizationService.java` | 批处理对比服务。接收多个显式配置，逐个运行仿真，按目标指标计算结果并返回对比列表；当前已弱化为 batch compare，不承担 MCTS 或遗传算法搜索职责。 |
| `service/QueueTheoryMetricsCalculator.java` | 排队论指标计算服务。在配置满足泊松到达、指数服务等条件时，生成理论排队指标供仿真结果对照。 |

## 13. 代码阅读建议

建议按以下顺序阅读源代码：

1. 先阅读 `SimulationApplication.java` 和 `SimulationController.java`，了解系统入口和对外接口。
2. 再阅读 `SimConfig.java`、`SimulationReport.java`、`SimulationSummary.java`，理解输入配置和输出报告结构。
3. 然后阅读 `SimulationRunService.java`、`SimulationConfigNormalizer.java`、`SimulationArrivalScheduler.java`，理解一次仿真如何被准备和启动。
4. 接着阅读 `SimulationEngine.java` 及 `engine` 包下各事件类，理解离散事件仿真的执行流程。
5. 最后阅读 `SimulationApiIntegrationTest.java` 和 `SimulationControllerTest.java`，通过测试用例确认接口契约和核心业务规则。
